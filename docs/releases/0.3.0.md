# Audio Path Inspector v0.3.0

This is a pre-release build focused on collecting more evidence when APO registration is not exposed through the usual endpoint properties.

## Highlights

- Added endpoint property store dump with audio/FX/APO related property grouping.
- Added registry evidence collection from MMDevices, PnP device, and endpoint property CLSID registry paths.
- Added audio/APO candidate highlighting for audiodg.exe loaded DLL diagnostics.
- Expanded the device details tree by default for easier scanning.

## Included from earlier releases

- Capture / Render endpoint switching.
- Registered APO inspection from endpoint properties.
- Inferred APO chain display grouped by Stream, Mode, and Endpoint effects.
- Windows Audio Effects inspection through `IAudioEffectsManager`.
- Audio Enhancements state detection from endpoint properties.
- Shared, Shared RAW, and Exclusive WASAPI open tests.
- APO notification interface support diagnostics.
- audiodg.exe loaded DLL diagnostics as supporting runtime evidence.
- Tree-based inspection details view.
- Device enumeration and endpoint inspection work runs off the GUI thread.

## Notes

- `APO Chain (inferred)` is based on endpoint property registration and is not a guaranteed runtime execution trace.
- Endpoint property names and values are best-effort diagnostics; unknown property keys are still shown by raw key.
- Registry evidence is read-only and may be incomplete when Windows denies access or expected registry paths are absent.
- Windows Audio Effects results only reflect effects reported by Windows APIs. Hardware DSP, USB device DSP, vendor-private processing, and driver-internal processing may not be visible.
- Some endpoints may not support `IAudioEffectsManager`, Shared RAW mode, or exclusive mode. In those cases the app displays the HRESULT or diagnostic message.
- audiodg.exe module enumeration may require sufficient process access and can fail on some systems; failures are shown as diagnostics.
