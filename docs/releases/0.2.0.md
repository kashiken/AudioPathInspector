# Audio Path Inspector v0.2.0

This is a pre-release build focused on making endpoint inspection more useful across capture and render devices.

## Highlights

- Added Capture / Render endpoint switching.
- Added registered APO inspection from endpoint properties.
- Added inferred APO chain display grouped by Stream, Mode, and Endpoint effects.
- Added Windows Audio Effects inspection through `IAudioEffectsManager`.
- Added Audio Enhancements state detection from endpoint properties.
- Added Shared, Shared RAW, and Exclusive WASAPI open tests.
- Added APO notification interface support diagnostics.
- Moved device enumeration and endpoint inspection work off the GUI thread.

## Notes

- `APO Chain (inferred)` is based on endpoint property registration and is not a guaranteed runtime execution trace.
- Windows Audio Effects results only reflect effects reported by Windows APIs. Hardware DSP, USB device DSP, vendor-private processing, and driver-internal processing may not be visible.
- Some endpoints may not support `IAudioEffectsManager`, Shared RAW mode, or exclusive mode. In those cases the app displays the HRESULT or diagnostic message.
